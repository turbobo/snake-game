# 贪吃蛇大作战 · 技术架构文档

> **版本**: 1.0.0 · **更新时间**: 2026-06-17 · **适用代码**: `index.html` (1624 行)

## 1. 项目概览

贪吃蛇大作战是一款基于 HTML5 Canvas 的 2D 实时竞技游戏，单文件交付（HTML + CSS + JS 全部内联），无需构建工具。支持单人（含 AI 对手）和在线 4 人对战两种模式，融合经典贪吃蛇玩法与道具系统、连击机制、里程碑奖励、长度速度缩放等现代要素。联机采用 Host-Authority + GoEasy PubSub 架构，Host 以约 15Hz 向所有客户端广播完整游戏状态，客户端本地预测自身蛇的移动以保证响应性。

## 2. 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 渲染 | Canvas 2D + requestAnimationFrame | 60fps 目标，`setTransform` 适配 devicePixelRatio |
| 输入 | Keyboard / Pointer Events / Touch | `pointerdown/up` 优先，降级 touch/mouse |
| 音频 | 无（未实现） | 预留 hook 点 |
| 联机 | GoEasy PubSub SDK v2.14.9 | CDN 加载，`hangzhou.goeasy.io` |
| 部署 | 纯静态单文件 | 任意静态托管（EdgeOne Pages / GitHub Pages / Vercel） |
| 字体 | 系统字体 | `-apple-system, BlinkMacSystemFont, PingFang SC, Microsoft YaHei`，不依赖 Google Fonts（中国网络慢） |
| 存储 | localStorage | 持久化 `snake-best3`（历史最高分） |

## 5. 游戏状态机

```
menu → (单人) → playing → dying → dead → menu
menu → (在线) → lobby → playing → dead → lobby / menu
playing ⇄ paused  (仅单人)
```

`gameState` 枚举：`'menu' | 'lobby' | 'playing' | 'dying' | 'dead' | 'paused'`。死亡后本地模式 900ms 延迟跳转 `dead`；在线模式由 Host 广播 `game-over` 触发。

## 6. 核心系统

### 6.1 蛇体数据结构

```js
{
  name, playerId,
  body: [{x,y}, ...],          // body[0] 是头
  dir: {x,y}, nextDir: {x,y},  // 当前方向 / 下一帧方向
  bodyColor, headColor,
  score, kills, combo, maxCombo,
  alive, dead, respawnTimer,
  isAI, moveTimer,
}
```

`kills/combo/maxCombo` 由 Host 维护并广播，客户端 HUD 从自己的蛇读取。

### 6.2 移动系统 (tickSnake)

每个 tick 内：
1. `sn.dir = sn.nextDir`（方向锁定）
2. 计算新头 `(nx, ny) = head + dir`
3. 碰墙：有护盾则 `(nx+cols)%cols` 穿越；否则死
4. 碰自己 / 碰其他蛇身：死亡（护盾撞别人则对方死）
5. 新头入队，`postSnakeMove` 处理吃食物 / 吃道具 / 磁铁

### 6.3 长度-速度缩放

```js
lenFactor = Math.min(2, 1 + (body.length - 3) * 0.03)
effSpeed  = baseSpeed * lenFactor     // 越长越慢
```

冲刺时再 × 0.5（最低 40ms/格）。

### 6.4 道具系统 (POWERUP_TYPES)

| 道具 | 颜色 | 持续时间 | 效果 |
|------|------|---------|------|
| ⚡ 加速 | 红 | 5000ms | 速度减半（更快） |
| 🛡️ 护盾 | 紫 | 5000ms | 穿墙、撞死对手 |
| 💎 双倍 | 金 | 8000ms | 食物积分 ×2 |
| 🧲 磁铁 | 绿 | 7000ms | 曼哈顿距离 ≤ 6 的食物向玩家靠近 1 格 |

地图最多 3 个道具同时存在，12s 后自动消失。每吃 4 个食物 / 达到里程碑会刷新新道具。

### 6.5 连击与里程碑

- **连击**: 2 秒内连续吃食物，combo++；≥3 连击触发倍率 `1 + min(combo, 8) * 0.2`
- **里程碑**: 50 / 100 / 200 / 350 / 500 / 800 / 1200 分触发屏幕震动 + 2 个道具奖励

### 6.6 AI 系统 (tickAI)

贪心算法：每 tick 在 4 个方向中选"不撞自己 + 不撞别人 + 曼哈顿距离食物最近"的方向。禁止 180° 掉头。

### 6.7 重生

`killSnake` 设置 `respawnTimer = 20`（约 3 秒）。`respawnSnake` 寻找 3 格连续空位（头 + 2 节身体），最多尝试 200 次。

## 7. 渲染系统

### 7.1 帧循环 (loop)

```
loop(time) →
  dt = min((time-lastTime)/1000, 0.05)   // 上限 50ms 防抖
  if playing → tick host/local/client
  shakeTimer 衰减
  updateParticles(dt)
  draw()
  updateScoreboard()
  requestAnimationFrame(loop)
```

### 7.2 绘制层级（从底到顶）

1. 背景 `#0d1117` + 网格线
2. 轨迹点 `trailPoints`（半透明小圆）
3. 道具（emoji + 径向发光）
4. 食物（径向渐变 + 高光）
5. 蛇体（圆角矩形，头带眼睛和名字标签）
6. 粒子爆发（吃食 / 死亡）
7. 飘字（积分、连击、里程碑）

### 7.3 插值动画

`interpProgress = moveTimer / effSpeed` 在 0~1 之间线性增长。`drawSnakeBody` 根据 `interp` 对头部向下一格、身体向上一节插值，避免跳格。

## 8. 在线多人

### 8.1 架构

- **Host-Authority**: 所有游戏逻辑（吃食物、碰撞、死亡、计分）只在 Host 跑
- **GoEasy PubSub**: `hangzhou.goeasy.io`，AppKey `BC-849dcab428524333a394bbc19744e72b`，房间 channel 形如 `snake-ABC123`
- **广播频率**: 66ms 一次 `game-state`（≈15Hz）
- **心跳**: Host 每 2s 广播 `room-info` 同步玩家列表

### 8.2 客户端预测

客户端在两次 game-state 之间按本地 `nextDir` 预测移动（只 unshift/pop，不吃不撞）。收到 `game-state` 时整体替换蛇体，但保留本地 `pendingDir` / `nextDir`，避免按键丢失。

### 8.3 消息协议

| 类型 | 方向 | 载荷 | 触发 |
|------|------|------|------|
| `player-join` | 客→主 | name, colorIdx | 客人加入 |
| `request-room-info` | 客→主 | — | 客人加入后立即请求 |
| `room-info` | 主→客 | players[] | Host 收到请求 / 每 2s 定时 |
| `direction` | 客→主 | dir {x,y} | 按键变化 |
| `game-start` | 主→客 | players, scoreTarget, baseSpeed | 房主点"开始游戏" |
| `game-state` | 主→客 | snakes[], foods, powerups, particles, floatingTexts | Host 每 66ms |
| `game-over` | 主→客 | reason {winnerId, winnerName}, scores[] | 达成目标分数 / 仅剩 1 人 |
| `player-leave` | 客→主 | name | 离开房间 |

### 8.4 错误处理

`formatGoEasyError(err)` 统一格式化 GoEasy 错误：
- 字符串 → 直接返回
- 对象 `{code, content}` → `[code] content`
- 对象 `{message}` → message
- 其他 → JSON.stringify

注意 GoEasy `connect()` 的 `data` 参数必须是对象（`{name}`），传字符串会触发 400 错误。

## 9. 输入系统

| 设备 | 方向 | 冲刺 | 暂停 |
|------|------|------|------|
| 键盘 | WASD / 方向键 | Shift | 空格（仅单人） |
| 触屏 | D-pad（76px） | ⚡ 圆形红键（按住） | 不支持在线暂停 |
| 触屏滑动 | 滑动方向 | 双击（仅单人） | — |

Pointer Events 优先，降级 touch/mouse。按下时触发 `navigator.vibrate` 触觉反馈（8ms 方向 / 12ms 冲刺）。

## 10. 移动端布局

- D-pad 使用 CSS Grid 显式定位（`.ctrl-up/left/down/right/sprint`），3 行 × 4 列
- 玻璃拟态：`backdrop-filter: blur(10px)`，背景半透明深蓝
- 按下态：缩放 0.94 + 霓虹发光 + 渐变背景
- `resize()` 在移动设备下预留 320px 底部空间，避免画布关键区域被遮挡
- 冲刺键 online 模式无实际效果（Host 未同步 boosting 状态），仅视觉反馈
